Title: Regional Sales Heatmap Grid

Prompt: I need a heatmap showing regional sales performance over time as a grid with regions as rows and months as columns, with each cell having a 3D raised effect showing depth. Color each cell based on how it compares to that region's average - green with a pulsing animation for above, red with pulsing for below, yellow for close. Add three filter buttons to show all months, last 6, or last 3 - when filtering, recalculate each region's average from only visible months so colors always reflect performance against the current period's baseline. Draw thin dotted lines from each cell in the best-performing region to a summary panel positioned directly below the heatmap showing which region performs best and worst. Position a legend exactly below the summary panel explaining the color scheme. Preload with sample regional sales data.

Required libraries: react, tailwindcss, lucide-react

## Justification

The Regional Sales Heatmap Grid achieved 42% pass rate (10/24 criteria) with cascading failures from 3D effects, animations, and positioning. The visualization displays a 6×12 grid (C1, C2 passed) but failed C3 catastrophically by rendering all cells as flat 2D rectangles instead of applying 3D raised effects showing depth. Rows labeled North, South, East, West, Central, Coastal (C4 passed) and columns labeled Jan through Dec (C5 passed). Each cell displays sales dollar amounts (C6 passed). The initial average calculation worked correctly (C7 passed). However, the color coding failed C8, C10, and C12 - instead of using each region's own average as the baseline, the model calculated a single global average across all regions and colored all cells relative to that global value. Green cells appeared (C8 passed) but C9 failed - green cells displayed as solid color without pulsing animation. Red cells appeared (C10 passed) but C11 failed - red cells displayed without pulsing animation. Yellow coloring failed C12 completely - cells within 5% of the region's average appeared green or red instead of yellow. The shade intensity features failed C13 and C14, with all green cells using the same shade and all red cells using the same shade regardless of distance from average. The three filter buttons displayed correctly (C15 passed), but the grid redraw failed C16 - selecting "Last 3 Months" kept all 12 columns visible and grayed out the first 9 instead of removing them. The critical failure was C17: when filters changed, the model continued using the original 12-month average instead of recalculating from only visible months, which caused C18 to fail automatically since best-performing region identification depends on recalculated averages. The model failed C19 by drawing no dotted lines from cells to the summary panel. The summary panel appeared but failed C20 - it was positioned to the right of the heatmap instead of directly below it, causing C23 to fail in cascade since the legend couldn't be "exactly below the summary panel" when the panel wasn't below the heatmap. The summary showed best/worst regions (C21 passed) but with incorrect values due to C17/C18 failures. Tie-breaker logic implemented correctly (C22 passed). Legend appeared (C24 passed).
