Category: Web Application

Prompt style: Detailed Specification

Title: DevOps Command Center

Prompt: Build a DevOps command center with three panels for managing server commands. Display a command form panel with two input fields: "Command" text input and "Server" dropdown with three options: "Production", "Staging", "Development", with required field validation showing red pulsing border if Command field is left empty. Display a command queue panel showing queued command cards with each card displaying the command text at top-left and a green "Execute" button positioned at top-right corner of the card. Display a terminal output panel showing a terminal-style display with dark background and green monospace text. When users fill both form fields with valid data and click the "Queue Command" button, add a new command card to the queue with the entered data and clear all form fields. When users click the green "Execute" button on any command card, remove that card from the queue, display the command in the terminal with a typing animation effect character by character, then after the animation completes display "SUCCESS" or "ERROR" output below the command with a fade-in animation. When users click the "Clear Terminal" button positioned at bottom-right of terminal panel, remove all text from the terminal display with a fade-out animation.

Required libraries: react, tailwindcss, lucide-react

## Justification

The DevOps Command Center will be evaluated based on its ability to manage server commands through form input with animated validation, command queue management with CRUD operations, and terminal output with character-by-character typing and fade animations. Success requires displaying a command form panel with Command text input showing red pulsing border if left empty, and Server dropdown with three options (Production, Staging, Development). The command queue panel must display command cards with command text at top-left and a green Execute button positioned at top-right corner of each card. The terminal output panel must show dark background with green monospace text. When users fill both form fields with valid data and click the Queue Command button, the implementation must add a new command card to the queue and clear all form fields. When users click the green Execute button on any command card, the implementation must remove that card from the queue, display the command in the terminal with typing animation character by character, then after animation completes display SUCCESS or ERROR output below the command with fade-in animation. When users click the Clear Terminal button at bottom of terminal panel, the implementation must remove all terminal text with fade-out animation. This task tests form-based data entry with dropdown selection and animated validation feedback, CRUD operations for queueing and executing commands, terminal styling with monospace fonts and dark theme, character-by-character typing animation with sequential timing, fade-in animation triggered after typing completion, and fade-out animation for clearing content.
