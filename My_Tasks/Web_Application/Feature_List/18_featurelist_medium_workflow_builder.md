Category: Web Application

Prompt style: Feature List

Title: Workflow Builder with Step Dependencies

Prompt: Build a workflow builder with three panels for managing workflow steps. Display a step form panel with three input fields: "Step Name" text input, "Priority" dropdown with three options: "High", "Medium", "Low", and "Assignee" text input with required field validation showing red pulsing border if left empty. Display a steps list panel showing pre-populated steps with each step card displaying step name at top-left and a blue "Edit" button positioned at top-right corner of the card. Display a tree hierarchy panel showing a parent node labeled "Project" at the top connected by straight lines to three child nodes labeled "Phase 1", "Phase 2", "Phase 3" positioned below in a horizontal row. When users fill all three form fields with valid data and click the "Add Step" button, add a new step card to the steps list with the entered data and clear all form fields. When users click the blue "Edit" button on any step card, populate the form fields with that step's data and change the "Add Step" button text to "Update Step". When users click any node in the tree hierarchy, apply orange pulsing background to that node and display the node's step count centered below the node name.

Required libraries: react, tailwindcss, lucide-react, react-beautiful-dnd

The Workflow Builder will be evaluated based on its ability to manage workflow steps through form input with animated validation, CRUD operations on step cards, hierarchical phase tree visualization with pulsing animations, and complex internal positioning. Success requires displaying a step form panel with three input fields: Step Name text input, Priority dropdown with three options (High, Medium, Low), and Assignee text input with required field validation showing red pulsing border if left empty. When users click the blue Edit button on any step card, the implementation must populate the form fields with that step's data and change the Add Step button text to "Update Step". When users click any node in the tree hierarchy, the implementation must apply orange pulsing background to that node and display the node's step count centered below the node name. This task tests form-based data entry with dropdown selection and animated validation feedback, CRUD operations for creating and editing workflow steps, form state management with button text changes, hierarchical tree visualization with straight line connections, animated pulsing effects for validation errors and node selection, and internal positioning complexity (top-left/top-right card positioning, horizontal row tree structure, centered-below count display).
