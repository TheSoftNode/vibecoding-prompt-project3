Title: Workflow Builder with Step Dependencies

Prompt: Build a multi-page workflow application with navigation. Display a dashboard view with "Step Name" text input, "Phase" dropdown with options "Phase 1", "Phase 2", "Phase 3" positioned at bottom-left, and red pulsing border on Step Name if empty. Display a steps view via "Steps" link showing step cards with Edit button at top-left corner. Display a phases view via "Phases" link showing "Project" parent node connected to three child nodes "Phase 1", "Phase 2", "Phase 3" in horizontal row. When users submit the form, add step card to steps view and make the connecting lines glow blue. When users click Edit in steps view, navigate to dashboard, populate form, and change button to "Update Step". When users click a phase node, apply orange pulsing background, display step count centered below node, and highlight all connecting lines from that node.

Required libraries: react, tailwindcss, lucide-react, react-beautiful-dnd

## Justification

The Workflow Builder works as a multi-page application with navigation, form validation with animated effects, CRUD operations with routing, and phase tree visualization. The dashboard view displays a Step Name text input with red pulsing border if empty and Phase dropdown with options (Phase 1, Phase 2, Phase 3) positioned at bottom-left. The steps view is accessible via "Steps" navigation link and shows step cards with Edit button at top-left corner. The phases view is accessible via "Phases" navigation link and displays a "Project" parent node connected to three child nodes "Phase 1", "Phase 2", "Phase 3" in horizontal row with connecting lines. When users submit the form, a step card is added to the steps view and the connecting lines glow blue. When users click Edit on a step card, the app navigates to the dashboard view, populates the form with step data, and changes the submit button text to "Update Step". When users click a phase node, orange pulsing background is applied, step count for that phase displays centered below the node name, and all connecting lines from that node are highlighted.
