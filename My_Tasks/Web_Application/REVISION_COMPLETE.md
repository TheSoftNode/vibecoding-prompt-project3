## 🎓 KEY COMPLEXITY PATTERNS

### **Pattern 1: Threshold-Based Cascades**
```
Value crosses threshold → Visual change → Badge/notification → Derived metric update
```
- Budget 80% → Yellow bar
- Budget 100% → Red bar → "Overspent" badge
- Meeting load 75% → Red header
- Timer < 5s → Red text

### **Pattern 2: Multi-Source Aggregations**
```
Multiple inputs → Calculation → Display → Real-time sync on any change
```
- All expenses → Category totals → Budget %s → Progress bars → Colors → Badges
- All question times → Average + Min → Analytics display
- All item prep times → Total prep time → "Est. Ready" display
- All events per day → Meeting load % → Header color

### **Pattern 3: Conditional Feature Activation**
```
State check → Feature unlock/display → Visual feedback
```
- Cart has combo → 15% discount → "Combo Deal!" badge → Green borders
- Subtotal > $500 → Tier 1 discount (or)
- Subtotal > $1000 → Tier 2 discount (higher wins) → "You saved $X"
- Spicy + Large drink → "Free Size Upgrade!" notification

### **Pattern 4: Time-Based State Evolution**
```
Time passes → State updates → Dependent features recalculate → Visual changes
```
- Days increment → Overdue check → Badge visibility
- Timer decrements → Color change at 5s → Auto-submit at 0s → Next question
- Streak increments → Milestone check → Card color change → Badge display
