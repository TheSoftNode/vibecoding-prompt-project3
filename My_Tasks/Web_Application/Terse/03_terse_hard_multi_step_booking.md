Category: Web Application

Prompt style: Terse

Title: Multi-Step Booking System with Conflicts

Prompt: Three-step booking form: Step 1 shows date picker and time slots. Gray out unavailable slots (12 PM tomorrow, 3 PM today). Step 2 shows service selection with prices and quantity inputs. Calculate estimated duration (each service = 30 minutes × quantity). Display dynamic end time (selected time + duration). Show red warning if end time exceeds 5 PM ("We close at 5 PM"). Apply 10% morning discount if selected time is before 11 AM, show original and discounted prices. Step 3 shows summary with date, time range (start-end), services, quantities, subtotal, discount if applicable, and final total. Next/Back buttons navigate steps. Summary updates when selections change. Confirmation page shows all booking details. On load, display step 1 with tomorrow's date selected and three time slots: 9 AM, 12 PM (grayed out), 3 PM. Services: Haircut $30, Coloring $50, Styling $25.

Required libraries: react, tailwindcss, date-fns, lucide-react
